Hugo博客搭建指南

2024.01.31

最近在折腾Hugo静态博客，但是碍于个人水平有限，期间查阅了大量资料进行了诸多尝试才初获成功。因此，在这里记录下整个过程以资备忘。

本文要达成的目的，配置 Hugo 通过 Github Pages 发布后再部署到 Vercel。所需要的环境依赖，Github 系列产品、Git、Hugo、Windows 11。

一、Github Pages 配置

对于 Github 这里就不再赘述，重点说一下需要创建的仓库及仓库配置。

首先需要在 Github 中创建两个仓库，其一为 public 仓库，用以发布网站；其二为 private 仓库，用以存储建站资源文件，如配置文档、主题、内容等。在本地使用 Markdown 编辑网站内容后，Push 到 Github 的 private 仓库，然后由 Github Actions 调用 Hugo 自动编译并推送到 public 仓库，最终由 Github Pages 部署网站。

1 创建 Public 仓库

此仓库用于 Github Pages 发布使用。仓库必须是 Public 性质的，仓库名称必须是 Github 账户名后面加上 .github.io 。如下图所示：

其他配置默认即可，仓库创建成功后 Github 会自动将该仓库发布到http://linkinecho.github.io 网址。

仓库创建成功后可以添加一个 index.html 文件，文件中随便写入内容，保存后等待 Github 后台渲染成功后即可访问。

2 创建 Private 仓库

根据设想，在本地创建编辑Markdown文件后使用Github Pages

设想是实现用 Markdown 文件编辑网站内容，然后使用Github的 Hugo 服务进行转换，自动生成静态网站，因此计划创建另一个仓库容纳 Hugo 建站必备的内容。

该仓库没有特殊的要求，可以是公有（Public）的，也可以是私有（Private）的，这里暂时采用私有仓库的方案，创建仓库的方法同前面创建公有仓库的方法基本没什么区别，只不过类型可以选择Private的，仓库名称也没有限制，我将其简单命名为 myHugoWebSite。
